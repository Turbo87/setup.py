Welcome to the setup.py cheatsheet!

This is an interactive guide for exploring various important parameters of the setup.py packaging script format for python applications.

You can access information about parameters by hovering over or clicking the property name.

Thanks to Nodejitsu for their package.json cheatsheet which inspired me to create this page.

The Python shebang

Use this in the first line of the script to tell Unix-like system to start it with the Python interpreter by default.

The distutils import

setup.py is a standard python script that calls a setup() function. You have to import the function from the distutils.core module before you can use it.

The setup() call

Most information that you supply to the Distutils package is supplied as keyword arguments to the setup() function.

Hover over or click on the individual arguments to see the details.

The unique name of your package

(Required)

This will indicate the name of the package in the Python Package Index (if you choose to publish it).

Keep it short and try not to use any special characters.

The package version

(Required)

This field describes the version of your package. It should use the format described by PEP386.

Try to follow the rules of Semantic Versioning.

Examples:

  • 0.4
  • 0.4.1
  • 0.5a1
  • 0.5b3
  • 0.9.6
  • 1.0
  • 1.0.4a4
  • 1.0.4b1

The description of the project

Try to keep it short, concise and in one line.

The project author

The name and email address of the package author.

The current project maintainer

The name and email address of the current package maintainer.

This should only be used if the information is different from the package author. For example for forks or projects that are no longer maintained by their original author.

Project classifiers

This list helps the Python Package Index to classify and filter projects by their most important attributes.

Only string from the official list must be used. Including custom classifiers will result in package rejection.
